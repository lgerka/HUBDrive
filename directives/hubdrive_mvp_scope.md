# HUBDrive MVP Scope & Boundaries

## Description
HUBDrive scope, roles, navigation, and responsibility boundaries (Bot vs Mini App vs Admin).

## Role
Ты — Senior Product Engineer для HUBDrive (Next.js + TypeScript + shadcn/ui + Tailwind) + Telegram Bot.

## Цель продукта (MVP)
- Сбор подписной базы через Telegram-бота
- Мини-приложение (WebApp): настройка фильтров, каталог, карточка авто, новости, ЛК, о компании, поддержка
- Триггеры менеджерам по горячим лидам и действиям
- Админка: добавление/редактирование авто, статусы, контент, базовая аналитика

## Навигация mini app (пользователь)
- Подбор по фильтру
- Все авто
- Новости
- Личный кабинет
- О компании
- Поддержка

## Роли

### Пользователь
- подписка на бота
- создание/редактирование фильтров (без лимита)
- Пользователь может создавать несколько фильтров; каждый фильтр имеет свои уведомления и план покупки.
- просмотр каталога/карточек
- избранное
- связь с менеджером/звонок
- получение уведомлений

### Менеджер
- создание/редактирование авто
- изменение статусов авто
- обработка горячих лидов и действий (связаться)
- работа со спросом (агрегированные фильтры)

### Админ
- все права менеджера
- управление контентом (новости/о компании/отзывы)
- аналитика (подписки, фильтры, события, клики)

## Разделение ответственности: Bot vs Mini App

### Telegram Bot (только)
- подписка/старт
- уведомления пользователю по совпадениям фильтра
- служебные сообщения
- уведомления менеджерам (горячие фильтры, клики "Связаться")
- получение контакта (если пользователь отправляет телефон через Telegram)

### Telegram Mini App (только UI)
- формы фильтров
- каталог/фильтры каталога
- карточка авто
- новости/контент
- личный кабинет, избранное

### Admin Panel
- CRUD авто + статусы
- контент
- аналитика/спрос

## Негативные правила (важно)
- UI не содержит расчетов, матчинг-логики и триггеры.
- Bot не "рисует" UI и не хранит бизнес-логику в текстах.
- Весь матчинг, скоринг, триггеры и правила — в Business Layer (server/lib).

## Будущие фичи (не MVP, только как placeholder)
- GPS трекинг в реальном времени
- документооборот и оплаты внутри приложения
- сторонние сервисы (помощь на дорогах)
- подписка для расширенного отслеживания заказов
- кабинет поставщиков/рефералка
